= Runbook Migrasi Baileys di Pterodactyl

== 1) Persiapan Server

. Buat server baru di panel Pterodactyl (Node.js egg).
. Upload folder `services/baileys-bot` ke server.
. Jalankan install dependency:
+
[source,bash]
----
npm install --omit=dev
----
+
. Set startup command:
+
[source,bash]
----
npm run start
----
+
. Siapkan storage persisten untuk auth state:
* Set `BAILEYS_AUTH_DIR=/home/container/.baileys-auth`
* Pastikan direktori tidak terhapus saat restart/redeploy.

== 2) Set Environment Bot

Karena egg Node.js mem-prefix command dengan `/usr/local/bin/`, simpan env bot di file runtime:

[source,bash]
----
cat > /home/container/baileys-bot/.env <<'EOF'
PORT=2020
BOT_API_KEY=<BOT_API_KEY>
SESSION_NAME=default
APP_WEBHOOK_URL=https://catat-wang.vercel.app/api/whatsapp/webhook
APP_WEBHOOK_SECRET=<sama dengan WAHA_WEBHOOK_SECRET app>
BAILEYS_AUTH_DIR=/home/container/.baileys-auth
LOG_LEVEL=info
EOF
----

Wajib di `.env`:

* `BOT_API_KEY`
* `APP_WEBHOOK_URL=https://catat-wang.vercel.app/api/whatsapp/webhook`
* `APP_WEBHOOK_SECRET=<sama dengan WAHA_WEBHOOK_SECRET app>`
* `SESSION_NAME=default`
* `BAILEYS_AUTH_DIR=/home/container/.baileys-auth`

Opsional di `.env`:

* `PORT=3101`
* `LOG_LEVEL=info`

== 3) Reverse Proxy HTTPS

Expose service via domain stabil HTTPS (contoh: `https://wa-bot.example.com`).
Jangan gunakan IP mentah sebagai `BAILEYS_BOT_BASE_URL` di production.

== 4) Cutover Blue/Green

. Biarkan WAHA lama tetap berjalan.
. Verifikasi bot baru:
* `GET /health` => 200
* `GET /session` => status berubah ke `WORKING` setelah scan QR
. Update env app production (Vercel):
* `WHATSAPP_PROVIDER=baileys`
* `BAILEYS_BOT_BASE_URL=<domain HTTPS bot>`
* `BAILEYS_BOT_API_KEY=<BOT_API_KEY>`
* `WAHA_SESSION_NAME=default`
* `WAHA_WEBHOOK_SECRET=<APP_WEBHOOK_SECRET>`
. Trigger redeploy lewat commit ke GitHub (sesuai alur CI/CD).
. Monitor 30-60 menit.

== 5) Rollback

Jika ada gangguan:

. Ubah env app:
* `WHATSAPP_PROVIDER=waha`
* restore `WAHA_BASE_URL`
* restore `WAHA_API_KEY`
. Redeploy via push commit.

== 6) Recovery Cepat

Jika status bukan `WORKING`:

. Hit `POST /session` action `disconnect`.
. Hit `POST /session` action `ensure`.
. Scan QR dari `/settings/whatsapp`.
. Cek log server Pterodactyl dan endpoint `/health`.

== 6.1) Monitoring + Alert (No.4)

. Aktifkan workflow GitHub `WhatsApp Provider Healthcheck` (sudah jalan tiap 5 menit).
. Tambahkan secret repo untuk alert:
* `WHATSAPP_ALERT_WEBHOOK_URL=<webhook Slack/Discord/Telegram bridge>`
. Pastikan vars/secrets provider sudah terisi:
* `WHATSAPP_PROVIDER`
* `BAILEYS_BOT_BASE_URL`
* `BAILEYS_BOT_API_KEY`

Jika healthcheck gagal atau session bukan `WORKING`, workflow akan:

* menandai run sebagai gagal, dan
* mengirim alert ke `WHATSAPP_ALERT_WEBHOOK_URL` (jika di-set).

== 6.2) Backup Auth Session (No.5)

Jalankan backup auth secara terjadwal agar tidak perlu scan ulang saat recovery.

Backup otomatis menggunakan GitHub Workflow `Baileys Auth Backup`:

* Schedule: harian `02:10 WIB` (cron GitHub `10 19 * * *` UTC).
* Mekanisme: call API Pterodactyl `files/compress` untuk `.baileys-auth`.
* Retensi: simpan `14` archive terbaru (bisa diubah via `BAILEYS_AUTH_BACKUP_KEEP`).

Set GitHub vars/secrets berikut:

* `PTERODACTYL_BASE_URL=https://panel.xbotzlauncher.site`
* `PTERODACTYL_SERVER_ID=12f2d726`
* `BAILEYS_AUTH_BACKUP_KEEP=14` (opsional, default 14)
* `PTERODACTYL_CLIENT_API_KEY` (secret)

Untuk restore cepat, pakai script ini di server:

[source,bash]
----
sh /home/container/baileys-bot/scripts/restore-baileys-auth.sh /path/to/archive-YYYY-MM-DDTHHMMSSZ.tar.gz
----

== 7) Rotate/Revoke Credential (Wajib)

Urutan rotasi:

. Revoke `VERCEL_TOKEN` lama, generate token baru.
. Revoke `Pterodactyl Client API key` lama, generate key baru.
. Generate `WAHA_WEBHOOK_SECRET` baru.
.. Simpan secret lama sementara di `WAHA_WEBHOOK_SECRET_PREVIOUS` (Vercel) selama masa transisi.
. Generate `BAILEYS_BOT_API_KEY` / `BOT_API_KEY` baru.
. Rotate legacy `WAHA_API_KEY` jika masih aktif dipakai fallback.

Langkah update:

. Update Vercel env production:
* `WAHA_WEBHOOK_SECRET=<baru>`
* `WAHA_WEBHOOK_SECRET_PREVIOUS=<lama>` (sementara)
* `BAILEYS_BOT_API_KEY=<baru>`
. Update `.env` bot Pterodactyl:
* `BOT_API_KEY=<baru>`
* `APP_WEBHOOK_SECRET=<WAHA_WEBHOOK_SECRET baru>`
. Restart bot Pterodactyl.
. Redeploy production app Vercel.

Verifikasi pasca-rotasi:

. `GET /health` bot harus 200.
. Status session bot tetap `WORKING`.
. Kirim pesan transaksi uji dan pastikan transaksi tersimpan.

Finalisasi setelah 24 jam stabil:

. Hapus `WAHA_WEBHOOK_SECRET_PREVIOUS` dari Vercel.
. Pastikan webhook dengan secret lama sudah ditolak (`401`).
