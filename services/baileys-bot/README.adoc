= Baileys Bot Service

Service gateway WhatsApp berbasis Baileys untuk CatatWang.

== Endpoint

* `GET /health`
* `GET /session`
* `POST /session` body: `{ "action": "ensure" | "disconnect" }`
* `POST /messages/send` body: `{ "to": "+62812..." | "62812...@c.us", "text": "..." }`

Semua endpoint butuh header `X-Api-Key: <BOT_API_KEY>`.

== Local Run

[source,bash]
----
npm install
cp .env.example .env
npm run start
----

== Environment

Lihat `.env.example`.

Nilai `APP_WEBHOOK_SECRET` harus sama dengan `WAHA_WEBHOOK_SECRET` di app agar verifikasi signature tetap kompatibel (`x-waha-hmac`, SHA-512).

== Backup Auth State

Folder auth penting: `BAILEYS_AUTH_DIR` (default `/home/container/.baileys-auth`).

Script backup:

[source,bash]
----
sh /home/container/baileys-bot/scripts/backup-baileys-auth.sh
----

Script restore:

[source,bash]
----
sh /home/container/baileys-bot/scripts/restore-baileys-auth.sh /home/container/backups/baileys-auth/baileys-auth-YYYYMMDD-HHMMSS.tar.gz
----

Variable opsional backup:

* `BAILEYS_AUTH_BACKUP_DIR` (default `/home/container/backups/baileys-auth`)
* `BAILEYS_AUTH_BACKUP_RETENTION_DAYS` (default `14`)

== Bad MAC Notes

`Session error: Bad MAC` sporadis pada Baileys/libsignal bisa muncul saat rotasi/re-sync session key dan diklasifikasikan sebagai `warning` selama alur utama tetap sehat.

Rujuk playbook operasional:

* `deploy/baileys/BAD-MAC-OPERATIONS.adoc`
